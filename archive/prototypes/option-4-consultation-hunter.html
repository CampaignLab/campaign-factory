Prototype 4 · Participation · every open UK consultation

Democracy runs on deadlines

What it is: a radar that watches every open UK consultation — national and local — and drafts cited responses before the windows close. How it uses an agent factory: deterministic feed adapters pour decision windows into a queue as they open; ranking and drafting agents run against each one in the cloud; every draft lands in a human inbox for approval, and nothing is ever submitted by a machine. This page walks the six rungs from a chat guess to machine-speed participation.

About this concept

User & problem

Any small advocacy org. Statutory decision windows open and close unheard — verified: no aggregator of UK local-authority consultations exists. This machine is the missing thing, which is both the pitch and the warning.

Where the agents multiply — and why it isn't theatre

Honest inversion of the other prototypes: deterministic adapters and deadline arithmetic do the impressive part; agents add judgement only where it's needed (relevance vs org goals, drafting, discrepancy disclosure). The factory rung is the truest “factory” of the four — triggered by external events, continuous, with the human inbox's edits feeding back into the ranker.

What's real on stage
LiveGOV.UK, Citizen Space and Go Vocal pulls — three keyless JSON APIs verified 11 Jul 2026; ranking; drafting
CachedBrent record, TfL Barnes lifecycle, Leicester TRO documents
Deterministicfeed adapters, deadline arithmetic, citation binding, submission blocking
Agenticrelevance ranking, drafting, discrepancy handling
Simulatedthe 1,400-window national board (per-platform feeds real; the 317-council crawl not built — labelled)
Risks & tensions

Consultation flooding and evidentiary laundering at machine speed. The Brent miss is the emotional engine; if rehearsal audiences don't react to it, fold this radar into the winning prototype as a single panel.

Build complexity & 10-minute fit

Medium-low: three adapters, triage, drafting. Most defensible live layer and the only prototype that's a real product in August. UI mirrors shadcn (Awake template tokens); production would use shadcn/ui React components directly.
