Campaign Factory — four storyboards

Four candidate narratives for the 10-minute agents demo. Same underlying machine, four different stories. The job of tomorrow's session is to pick one.

Locked: user is a single campaign staffer
Demo length: 10 min (15 max)
Public data only · read-only tools · citations required
Human approval before anything exports
Full pre-cached fallback mode
How to read these: each storyboard is three beats — the setup, the live "wow" moment, and the factory reveal — with the presenter's script line under each screen. All screen data is synthetic but modelled on verified-accessible public sources; every mockup marks what would be LIVE, PRE-CACHED, or SIMULATED on stage. Options 1–3 share the same Leicester school-street data spine, so most build work transfers between them. Option 4 is the deliberate outlier.
OPTION 1

The Scout

The only campaigner at a small road-safety charity types a postcode. The machine finds the local fight, builds the evidence pack, drafts the campaign. Reveal: every school street in England lights up.

Fear Message: capacity
OPTION 2

The Ward Machine

A party field organiser is assigned a target ward they've never visited. The machine finds the issue that moves voters there and writes the doorstep script. Reveal: every battleground ward in the country, manufactured centrally.

Fear Message: volunteer displacement
OPTION 3

One Street, Many Masters

One set of facts about one Leicester street. Toggle the organisation and watch four opposed campaigns emerge from identical evidence. Reveal: all four side by side — whose campaign is real?

Fear Message: synthetic campaigning
OPTION 4

The Consultation Hunter

The machine watches every live public consultation in the country, ranks the ones its owner should care about, and drafts responses before deadlines close. Reveal: democratic feedback channels, flooded at machine speed.

Fear Message: evidentiary laundering

Decide, don't collect

The failure mode of this session is "can we do all four?" We can't — 10 minutes, one narrative, scarce build time. The comparison page scores all four on the ten criteria and has a blank row for your verdict.

What's already decided (don't reopen without cause): the user is one under-resourced campaign staffer, not a volunteer and not a big-org head office · the demo shows successive capability layers (prompt → tool-using agent → workflow → specialist agents → orchestration → factory) · agents only where they add value; geography, ingestion, citations and workflow state are deterministic code · a human approval gate that visibly blocks export.

What tomorrow decides: which story (1–4) · issue-advocacy vs electoral framing · which direction the factory scales (many places × one org, or one place × many orgs) · how frightening the ending is allowed to be.
