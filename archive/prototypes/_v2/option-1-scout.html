Presenter

About this concept

The Scout — enter a postcode, get the local issue, evidence, and campaign draft.

User & problem

The sole campaigner at a small road-safety charity. Finding a genuine, actionable local issue currently takes a week of council-website archaeology; this compresses it to minutes, with citations.

Central demonstration idea

One question — “what should we campaign on here?” — replayed against six escalating machines: model only → tools → verification workflow → specialist agents → human-gated orchestration → national factory.

Main flow

Postcode → geography + officeholders → candidate issues with citations → verified evidence pack → challenged claim corrected → drafted pack behind an approval gate → factory view.

What's real

Livepostcode/MP resolution; claim re-check; draft synthesis (on stage: replayed from cache)
Cachedcouncil documents, context datasets, national scan (all sources verified open, 11 Jul 2026)
Deterministicgeography joins, document fetch/parse, citation binding, approval state
Agenticissue ranking, contradiction finding, drafting
Simulatedthe 412-site factory run (real places, fake execution — labelled)

Benefits

Legible in 30 seconds; sympathetic protagonist; lowest ethical risk; the pipeline is the real recommended build.

Risks & tensions

The least frightening reveal — capacity, not menace. The panel must supply the unease. Risk of reading as “ChatGPT with better sources.”

Build complexity

Medium: full pipeline but one location. All critical sources verified keyless. UI kit mirrors Base UI component vocabulary; production would swap to @base-ui-components/react one-to-one.

In a 10-minute demo

Works as the clearest teaching version of the ladder. Weakest closing note of the four — pair with a strong panel handoff, or use the hybrid ending.
