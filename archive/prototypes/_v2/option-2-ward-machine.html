Presenter

About this concept

The Ward Machine — builds a ward brief and doorstep script for any target ward.

User & problem

A party field organiser assigned a target ward with no local members. The machine substitutes for local knowledge — which is exactly the capability and exactly the threat.

Central demonstration idea

Issues chosen for electoral utility, not local demand. The machine finds the wedge, writes the doorstep script, invents social proof — and a review agent plus a human catch it. Ends on volunteer displacement at national scale.

Main flow

Target ward → electoral brief (real 2023 results, real boundary fault line) → issues ranked by synthetic “resonance” → doorstep script with flagged invented claim → audience frames → approval gate → 120-ward battleground view.

What's real

Liveward/electoral resolution; ranking; script drafting (on stage: replayed from cache)
Cachedcouncil documents, 2023 ward results, CC0 councillor data — all verified open 11 Jul 2026
Deterministicjoins, citation binding, approval state, banned-tactic checks
Agenticissue ranking, frame selection, drafting, the flag
Simulated“resonance” methodology (disclosed in-product) and the 120-ward run (real ward names)

Benefits

Exactly matches the audience; the only option where the fear is native to the story; the flag-and-fix beat is the strongest approval moment of any prototype.

Risks & tensions

Ethically hottest: a real school's safety scheme instrumentalised for electoral gain, with real incumbents named. Party stays unnamed. Must be staged as “look straight at what's coming”, never a pitch. Requires Ed + Hannah's explicit sign-off.

Build complexity

Medium-high: option 1's spine plus electoral layer (verified open). Kit mirrors Base UI; production swaps to @base-ui-components/react.

In a 10-minute demo

Highest stage impact and panel fit. The price is the ethics cell — a deliberate choice, not an accident.
